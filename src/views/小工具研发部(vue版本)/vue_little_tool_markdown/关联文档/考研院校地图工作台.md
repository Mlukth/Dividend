# 考研院校地图工作台

一个基于 Vue 3 的中国地图交互式考研院校信息采集与管理工具。通过中国地图总控界面按省份录入院校，支持引导式三步表单填写、塔科夫式多窗口对比、PDF 导出、地理编码定位和本地文件存储。

## 项目简介
- **用途**：地毯式采集全国公办双非及以上层次的研究生院校信息，涵盖考试科目、分数线、报录比、导师信息、软信息（歧视双非/宿舍/课题组）及证明材料
- **核心特色**：地图右键省份交互 + 塔科夫式多浮窗对比模式，可同时打开多所院校窗口自由拖拽排列对比字段
- **数据持久化**：结构化数据存入 localStorage，支持 JSON 导入/导出备份，File System Access API 写入本地文件夹存储证明材料

## 技术栈
- 前端框架：Vue 3（Composition API `<script setup>`）
- UI 库：Element Plus（el-dialog / el-drawer / el-tag / el-select / el-upload / el-dropdown）
- 地图：ECharts v5/v6 + DataV CDN 全国 GeoJSON（`https://geo.datav.aliyun.com/areas_v3/bound/100000_full.json`）
- PDF 导出：jsPDF + html2canvas（通过浏览器渲染中文 HTML 后切片嵌入 PDF）
- 地理编码：高德 Web 服务 API（`/v3/geocode/geo`）
- 本地存储：File System Access API（`showDirectoryPicker`）

## 安装与运行
1. 确保已安装 Node.js（≥16）
2. 将 `考研院校地图工作台.vue` 放置于 Vue 3 项目（如 Vite 搭建）的 `src/views/` 目录下，项目动态路由会自动注册
3. 安装依赖：
   ```bash
   npm install echarts element-plus jspdf html2canvas
   ```
4. 配置高德地图 Web 服务 Key（文件内 `AMAP_KEY` 常量，已预填）
5. 启动项目访问对应路由即可

## 界面与操作说明

### 地图总控界面
- **中国地图**：自动加载 DataV 全国 GeoJSON（首次缓存至 localStorage），省份上有颜色深浅表示院校密度，标注 `省份名称 N` 格式
- **悬浮提示**：鼠标悬浮省份显示该省已录院校名称列表（bullet 格式）
- **散点标记**：有经纬度的院校以蓝色 pin 图钉标记在地图上，点击直接打开浮窗
- **右键菜单**：省份上右键弹出「新增院校」和「查看已记录」两个操作
- **省份标签**：底部显示所有已录省份的标签（省份名 + 数量），点击打开该省抽屉列表

### 顶部搜索
- 支持按院校名、省份、城市、专业、标签、考试科目关键字实时筛选
- 搜索结果即时反映在地图渲染和省份标签上

### 院校列表抽屉
- 右侧滑出，显示选中省份下所有已录院校
- 每项展示院校名、标签、专业、方向、城市
- 操作按钮：`对比`（打开浮窗）、`详情`（打开简历视图）、`删除`

### 引导式三步表单
- **步骤 1 — 基本信息**：院校名称（触发自动地理编码）、所在省、城市、专业、方向、标签（支持自定义标签）、照片上传
- **步骤 2 — 考试信息**：考试科目、分数线、报录比、学费/学制、备注
- **步骤 3 — 软信息与证明**：导师信息、复试风格、歧视双非、宿舍条件、课题组氛围、证明材料（文件上传 + 链接粘贴，支持关联到具体字段）

### 信息完整度
- 表单顶部实时计算字段填充率（含标签加分、照片加分、证明材料加分）
- 四段颜色指示：完整（绿）/ 过半（蓝）/ 起步（黄）/ 待填（灰）

### 塔科夫式多窗口对比
- 从抽屉点「对比」或直接点院校名 → 打开浮窗卡片
- 地图 pin 图钉点击 → 打开浮窗
- 浮窗可在页面内自由拖拽，支持多层叠加，点击提到最前
- 标题栏按钮：`—` 最小化、`☰` 打开完整简历详情、`✎` 编辑、`✕` 关闭
- 浮窗内容：照片、省市、标签、坐标、专业/方向/科目/分数线/报录比/学费/歧视双非/宿舍条件、备注摘要

### 简历视图
- 完整的院校信息展示，分区：照片 + 基本信息 / 考试信息 / 软信息 / 证明材料
- 包含信息完整度进度条
- 证明材料可点击链接打开、显示本地文件路径
- 底部按钮：导出 PDF、关闭

### 地理编码与校验
- 输入院校名 + 城市后自动调用高德 API 获取经纬度
- 自动对验收到的省份是否与填写省份一致，不一致弹出警告

### File System Access API
- 支持 Chrome/Edge 选择本地文件夹用于存放证明材料
- 保存院校时自动将上传文件写入选定文件夹
- 不支持该 API 的浏览器静默降级，不影响其他功能

### PDF 导出
- **模式一：单校详细报告** — 在简历视图点击「导出 PDF」，生成 A4 竖版报告，包含所有字段和证明材料
- **模式二：全局汇总总表** — 工具栏 PDF 下拉菜单选择，生成 A3 横版表格，14 列一校一行横向对比

### 备份与恢复
- 工具栏 ↓ 导出 JSON 备份
- 工具栏 ↑ 导入 JSON 恢复（带确认弹窗）

## 主要数据结构

```javascript
const entry = {
  id: String,           // 院校 ID（时间戳生成）
  name: String,         // 院校名称
  province: String,     // 所在省
  city: String,         // 城市
  coords: [lng, lat],   // 经纬度
  photo: base64|null,   // 照片（data URL）
  major: String,        // 专业
  direction: String,    // 方向
  tags: Array,          // 标签数组
  subjects: String,     // 考试科目
  scoreLine: String,    // 分数线
  ratio: String,        // 报录比
  tuition: String,      // 学费/学制
  remark: String,       // 备注
  tutorInfo: String,    // 导师信息
  interviewStyle: String, // 复试风格
  discrimination: String, // 歧视双非
  dormitory: String,    // 宿舍条件
  groupAtmosphere: String, // 课题组氛围
  subjective: String,   // 其他主观信息
  proofs: [{            // 证明材料数组
    type: 'file'|'link',
    label: String,
    data: String|File,
    field: String,      // 关联字段
    fsPath: String      // FSA 本地路径
  }]
}
```

## 主要函数说明

| 函数名 | 作用 |
|--------|------|

| `openDetailView(college)` | 打开简历视图对话框 |
| `goEditFromDetail()` | 从简历视图跳转到编辑表单 |
| `editCollege(college)` | 将院校数据填充到表单，打开编辑模式 |
| `saveCollege()` | 保存院校数据，处理 FSA 文件写入，更新 localStorage |
| `deleteCollege(college)` | 带确认弹窗的删除操作 |
| `resetForm()` | 重置表单所有字段到初始值 |
| `autoLocate()` | 调用高德 API 地理编码解析院校经纬度，含省份校验 |
| `handlePhotoUpload(uploadFile)` | 处理照片上传，转 base64 data URL |
| `handleProofFile/AddProofLink/RemoveProof()` | 证明材料增删操作 |

## 注意事项
- 高德 Web 服务 Key 已在文件中预填，该 Key 类型为「Web 服务」而非「JS API」，仅用于后端式 HTTP 请求
- ECharts 地图 GeoJSON 首次从 DataV CDN 加载后缓存至 localStorage（key: `china_geojson`），后续无需联网
- 院校结构化数据存储在 localStorage key `colleges_data` 中
- File System Access API 仅 Chrome/Edge 支持，Firefox/Safari 静默降级
- PDF 导出依赖 html2canvas 浏览器字体渲染中文，无需内嵌字体文件
- 照片以 base64 data URL 存储在 localStorage，大量高分辨率照片可能导致存储超限
